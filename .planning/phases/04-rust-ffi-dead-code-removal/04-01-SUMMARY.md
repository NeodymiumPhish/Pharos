## Decisions Made
- Kept ipnetwork and mac_address despite no direct `use` -- they are sqlx feature dependencies needed for PostgreSQL type decoder registration
- Used `#[allow(clippy::not_unsafe_ptr_arg_deref)]` on the ffi module declaration in lib.rs rather than marking all 35 functions as `unsafe extern "C"` -- the latter is a larger refactor with no ABI impact
- Left 9 pre-existing clippy warnings untouched (manual_strip, needless_borrows_for_generic_args, derivable_impls) as they are out of scope for this dead code removal phase
