<objective>
Fix the bug where SQL queries returning multiple columns with the same name (e.g., `SELECT count(*), count(*) FROM table`) display identical values for all same-named columns. The root cause is that rows are serialized as JSON objects keyed by column name, causing duplicate keys to overwrite each other. The fix changes all row data from name-keyed dictionaries to index-based arrays, from Rust serialization through to every Swift consumer.

Purpose: Correct data display for any query returning duplicate column names.
Output: Working index-based row data pipeline from Rust FFI through display, sort, filter, find, copy, export, and inspector.
</objective>
