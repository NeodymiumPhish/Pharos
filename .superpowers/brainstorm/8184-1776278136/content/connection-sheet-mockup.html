Behavior

  • Disabled state — shows "Test connection first" and is grayed out until Test Connection succeeds.
  • After test — populates with "None" + all available schemas from the database. User picks one.
  • "None" option — equivalent to "All Schemas" / no default. This is the default selection.
  • Editing existing connection — if a default schema was previously saved, it's pre-selected. Test Connection refreshes the list.
  • Saving without testing — the field stays disabled; no default schema is saved (preserves any existing default from prior edits).
